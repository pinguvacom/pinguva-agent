# Pinguva Agent Source 0.2.3

Этот каталог содержит отдельный публикуемый исходный код агента Pinguva.

Цель:
- показать пользователям, что агент прозрачен по коду;
- дать отдельный каталог, который можно выложить в отдельный GitHub-репозиторий;
- не смешивать агентский исходник с серверной частью.

Pinguva как SaaS-платформа, её backend и сервисная логика остаются закрытым платным продуктом.
Этот каталог публикуется только для прозрачности и аудита кода агента со стороны клиентов и security officers.

## Что агент делает

Агент:
- собирает системные метрики сервера;
- отправляет их в Pinguva по исходящему HTTPS;
- не требует входящих портов;
- не получает удалённый shell;
- не отправляет содержимое конфигурационных файлов;
- для контроля изменений конфигурации отправляет только metadata summary.

Это можно проверить прямо по исходникам в этом каталоге.

## Что агент не делает

Агент:
- не принимает входящие подключения;
- работает только через исходящий HTTPS;
- не выполняет удалённые команды;
- не отправляет содержимое конфигурационных файлов;
- не раскрывает код backend-части Pinguva, потому что backend в этот архив не входит.

## Коротко о Security и Privacy

Что агент собирает:
- системные метрики сервера: CPU, память, диск, сеть, uptime, ping;
- для Linux: Disk I/O, watched services, контроль изменений конфигурации;
- metadata по watched config profiles:
  - путь;
  - тип записи;
  - факт существования;
  - время изменения;
  - размер;
  - SHA-256.

Что агент не собирает и не отправляет:
- содержимое конфигурационных файлов;
- удалённые shell-команды;
- входящие сетевые сервисы;
- произвольные файлы с сервера.

Подробности вынесены в:
- `SECURITY.md`
- `PRIVACY.md`

## Что агент собирает

Основное:
- CPU, память, диск, сеть, uptime, ping;
- для Linux: Disk I/O, watched services, контроль изменений конфигурации;
- для Windows: базовые системные метрики.

Контроль изменений конфигурации для Linux отправляет только:
- путь;
- тип записи;
- факт существования;
- время изменения;
- размер;
- SHA-256.

Содержимое файлов не отправляется.

## Сборка

Linux/macOS:

```bash
go build -o pinguva-agent .
```

Windows:

```bash
GOOS=windows GOARCH=amd64 go build -o pinguva-agent.exe .
```

## Запуск

Одноразовый отчёт:

```bash
./pinguva-agent \
  --server https://example.pinguva.com \
  --token YOUR_AGENT_TOKEN \
  --once
```

С enrollment token:

```bash
./pinguva-agent \
  --server https://example.pinguva.com \
  --enrollment-token YOUR_ENROLLMENT_TOKEN
```

В реальной установке используется рабочий URL вашей инсталляции Pinguva или облачного контура, который выдаётся отдельно.

## Переменные окружения

- `AGENT_SERVER`
- `AGENT_TOKEN`
- `AGENT_ENROLLMENT_TOKEN`
- `AGENT_NAME`
- `AGENT_STATE_PATH`
- `AGENT_SERVICE_NAME`

## Структура

- `main.go` — запуск, регистрация и отправка отчётов
- `system_linux.go` — Linux-метрики, сервисы, конфиги
- `system_windows.go` — Windows-метрики
- `service_windows.go` — Windows service mode
- `service_other.go` — обычный режим для non-Windows
